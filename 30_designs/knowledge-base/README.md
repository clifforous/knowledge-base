# Knowledge Base Designs

## Purpose

This directory contains solution designs for the structure, tooling, and collaboration
model of the knowledge base itself.

These documents describe proposed or approved future behavior. They do not describe the
current repository structure unless they explicitly say so.

## Documents

- [Federated Personal and Canonical Knowledge Workflow](federated-personal-and-canonical-knowledge-workflow.md)
  - Status: proposed
  - Defines the future personal, peer, and canonical knowledge model and the proposed
    cross-agent kb integration.

## Boundaries

- Current repository procedures remain under ai/docs/.
- Current durable product and system knowledge remains under 50_knowledge/.
- Implementation plans and active work tracking do not belong in this directory.
- The existing knowledge base must be reorganized and exercised before the kb tool
  design is treated as implementation-ready.
