## Interface Domains

### Client-facing domain
Supports product-facing interaction flows:
- player profile and progression views
- team lifecycle actions
- gauntlet browsing and standings
- history and summary retrieval
- wallet and eligibility interactions

### Server-ingest domain
Accepts trusted event streams generated by gameplay or server contexts for competition telemetry and state progression.

This domain now also includes the gauntlet-stage runtime contract for:
- dedicated server stage status reporting
- dedicated server final stage results reporting
- runtime enforcement inputs for gauntlet stages

### Admin/operations domain
Supports privileged controls for:
- operational corrections and maintenance tasks
- sponsor and prize administration
- synchronization and cleanup operations
- gauntlet stage attempt inspection and intervention

## Permission Model
- Distinguishes user-context operations from privileged service-context operations.
- Enforces role-appropriate access at operation boundaries.
- Prevents misuse of user-self operations from privileged contexts.
- Should distinguish trusted dedicated-server writes from normal client traffic for gauntlet stage status and result publishing.

## Integration Boundary
Eventun orchestrates competition flow and delegates accounting transition execution to [[../accountun]].

## Current Notes
- Gauntlet stage sessions are currently public in AccelByte for simplicity.
- Public session visibility is not authorization.
- The dedicated server is the final admission gate for gauntlet stages.
- Eventun is the authoritative owner of stage-attempt identity and accepted outcomes.
